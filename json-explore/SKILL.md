## 触发条件

触发（满足任一）:
- 系统提示 MCP 工具返回过大、输出被写入文件
- 用户提供了 JSON 文件的路径，且该文件可能很大
- 用户说 "帮我看看这个 JSON"、"探索一下这个文件"、"看看结构"
- 上下文中有大 JSON 文件需要分析（尤其是 MCP 返回值）

不触发:
- 小 JSON 文件（不以文件行数判断，以文件大小是否小于5kb来判断），直接用 Read 工具即可
- 用户明确指定要用 jq / cat / less 等其他工具
- 非 JSON 格式文件（脚本会自动检测并提示，不需要 skill 层拦截）

歧义处理:
- 不确定文件大小时，先用 `wc -c` 检查，>5KB 则触发
- 无法判断是 JSON 还是其他格式 → 触发，脚本内置了非 JSON 检测和引导

## 行为准则

1. ❗ **禁止一次性读取大 JSON 文件内容** — 每次输出前自检：是否在引导用户逐步探索而非全部 dump
2. ❗ **从概览到细节，渐进式** — 先展示 key 树，再按需搜索/钻入/展开
3. ❗ **搜索结果路径直接可用于下一步** — `--find-key` 和 `--grep` 输出的路径格式与 `--path` 兼容，可直接复制使用
4. ❗ **禁止手动提取 MCP 嵌套 JSON** — 脚本已内置 `deep_expand` 自动展开 JSON 字符串，无需写临时 Python 脚本做二次解析

## 工具优先级（不可自行降级）

| 操作 | 首选工具 | 降级条件 | 降级工具 |
|------|---------|---------|---------|
| JSON 渐进式探索 | `json_explore.py` | 返回 "不是合法的 JSON" | 按脚本提示的工具分别处理 |

## 输出约束

禁止输出:
- 大段 JSON 原始内容（始终引导用户用 json_explore.py 渐进式查看）
- "让我来看看这个 JSON..." 等开场白（直接调用工具）
- 已知结构的复述（已经展示过的 key 树不再逐行重复）

## 执行流程

### Step 1: 结构概览

```bash
python3 json_explore.py <file>  # json_explore.py 与本 SKILL.md 同目录
```

首次默认展示：所有 key 的树形结构 + 叶子值前 50 字符 + 内容长度。
数组超过 5 项自动截断，只展示前 5 项 + 类型汇总。
**MCP 嵌套 JSON 自动展开**：如果值本身是 JSON 字符串（如 MCP 返回的 `text` 字段），脚本会自动递归解析，无需手动提取。

如果输出仍然太多，用 `-m 30 -a 3` 进一步收窄预览。

完成性要求:
- 已识别 JSON 的顶层结构
- 用户知道了哪些 key 存在、哪些区域值得深入

✅ Checkpoint: `Step 1 完成: 用户已了解顶层结构，可指定下一步钻入目标`

### Step 2: 按需深入

用户可能的需求和对应命令：

| 需求 | 命令 |
|------|------|
| 搜索某个 key 在哪 | `--find-key <key名>` |
| 搜索内容包含某字符串 | `--grep "<内容>"` |
| key + 内容组合筛选 | `--find-key <key> --grep "<内容>"` |
| 数组元素横向对比 | `-p '<路径>' -K code,type,value` |
| 钻入子树 | `-p '<路径>'` |
| 完整输出叶子值 | `-p '<路径>' --full` |
| 只看前 N 字符 | `-p '<路径>' -H N` |
| 前后各截一段 | `-p '<路径>' -H 200 -t 100` |
| 搜索范围限定在子树内 | `-p '<路径>' --find-key <key>` |

典型工作流：
```
第1次：默认概览 → 找到感兴趣的区域
第2次：--find-key / --grep 搜索定位，或 -p 数组路径 -K 字段列表 横向对比 → 拿到精确路径
第3次：-p '路径' --full → 获取完整值
```

完成性要求:
- 每次只执行一个查询动作
- 每次基于上一次的输出决定下一步
- 不预判用户想看什么，一步步来

✅ Checkpoint: `Step 2 完成: 用户获得了所需的具体值/子树`

## 依赖链

- Step 2 依赖 Step 1 中用户选定的钻入目标
- 每次调用之间不缓存结果，每次独立执行

## 事实性约束

所有结论必须引用来源（文件路径 + 具体输出行）。

| 场景 | 正确输出 | 禁止输出 |
|------|---------|---------|
| 脚本返回 "不是合法的 JSON" | 按提示引导用户用对应工具 | 继续尝试用 json_explore.py 解析 |
| --find-key 无结果 | "未找到匹配的 key，尝试换个搜索词" | 编造 key 路径 |
| --grep 无结果 | "未找到匹配的内容" | 猜测内容可能在哪 |
| 路径无效 | 展示脚本返回的可用 key 列表 | 凭记忆猜测路径 |

标注:
- 脚本输出 → 无标注
- 推测路径 → "⚠️ 尝试路径"
